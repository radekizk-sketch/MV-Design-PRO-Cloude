# CATALOG-FIRST READINESS TRUE REDESIGN

## Zasada
Readiness nie łata domyślnego create-first, tylko:
- topologię,
- zasilanie,
- stany normalne,
- braki migracyjne,
- niespójności analityczne.

## Bieżące zmiany
- Rozszerzono mapowanie blockerów do kodów domenowych i fix-actions.
- Braki katalogowe dla nowych obiektów przenoszone są na etap tworzenia (gates).
